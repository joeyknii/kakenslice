# Branching Workflow

Use separate branches for each type of update:

- `feature/images-*` for image/content updates
- `feature/order-flow-*` for customization and request flow changes
- `feature/enrollment-*` for class enrollment updates

Recommended process:
1. Create a new branch from main.
2. Make changes and commit.
3. Open a PR in GitHub.
4. Merge after review.
